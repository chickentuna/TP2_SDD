\part{Présentation du projet}

  L'objectif de ce TP est de coder un système capable de gérer et de parcourir des arbores par lien vertical et lien horizontal. La structure de donnée sera codée en C et le code contiendra les fonctions de recherche et suppression (avec élagage), ainsi que la possibilité de charger un arbre en mémoire à partir de son écriture algébrique dans un fichier. Le code pourra aussi afficher à l'écran le nombre de n?uds, le nombre de feuilles, et la hauteur d'un arbre.

\section{Description du projet}
\subsection{Schéma de la structure de donnée}
	Chaque n?ud d'un arbre est représenté par un mot de 3 bloc. Ce bloc contient: \begin{itemize}
\item La valeur du n?ud (ici, un seul caractère).
\item Le lien horizontal du n?ud, c'est à dire un pointeur vers son frère suivant (s'il en a un).
\item Le lien vertical du n?ud, c'est à dire un pointeur vers son premier fils (s'il en a).
\end{itemize}
Le premier n?ud est appelé \emph{racine} de l'arbre.\\
\par Le parcours d'un arbre se fait grâce à une pile. Une pile est représentée par un tableau d'éléments avec indice de tête.\\
\par Dans le code, les piles et les arborescences utilisées sont des \verb#struct#s. De plus, les éléments de la pile sont déclaré avec un typedef pour plus de généricité et son, par défaut, des \verb#void*#. Quant au valeur des noeuds d'un arbre, ce sont des \verb#char#.

\subsection{Organisation du code source}

Le code source est divisé en 3 parties : Le noyau (toutes les fonctions répondant au besoins du TP),\\ la section 'tests' pour les fonctions de test et leur système de gestion,\\ et le programme principal, \verb#main.c#, contenant la fonction \verb#main()# qui charge un fichier dont le nom est donné en ligne de commande et affiche la structure de donnée générée. Le fichier doit contenir la notation algébrique d'un arbre avec un seul caractère par élément.
Une exécution du programme avec le fichier \verb#exemple.arbre# vu par ddd est donné en annexe, ainsi que son rapport valgrind.

dans \verb#main.c#, qui permet soit d'exécuter les tests de toutes les fonctions du noyau, soit de charger un fichier à traiter (le nom du fichier doit être donné en commande).\\

\par Les macros et types généraux sont déclarés dans \verb#lib.h#.\\
Les fonctions relatives à la pile sont codés dans \verb#pile.c#.\\
\begin{itemize}
	\item \verb#initPile()#
	\item \verb#empiler()#
	\item \verb#depiler()#
	\item \verb#vide()#
	\item \verb#pleine()#
	\item \verb#libererPile()#
\end{itemize}
\newline
Les fonctions relatives aux arbres sont codés dans \verb#arbre.c#.\\
\begin{itemize}
    \item \verb#initArbre()#
    \item \verb#arbreSupprimer()#
    \item \verb#arbreSupprimerValeur()#
    \item \verb#arbreRecherche()#
    \item \verb#compterNoeuds()#
    \item \verb#mesurerHauteur()#
    \item \verb#compterFeuilles()#
    \item \verb#libererArbre()#
\end{itemize}
La fonction \verb#initArbre()# utilise 3 sous programmes : 
\begin{itemize}
    \item \verb#obtenirValeur()#
    \item \verb#obtenirOperation()#
    \item \verb#obtenirSuivant()#
\end{itemize}
\newline
La documentation des ces fonctions se trouvent respectivement dans \verb#pile.h# et \verb#arbre.h#. Ces fichiers contiennent aussi des fonctions \verb#toString()# pour convertir les structures de données associées en chaine de caractères, utilisées uniquement par le système de test.
\\
Les fonctions qui gèrent l'exécution et l'affichage des sorties des test sont déclarés dans \verb#testlib.h# et codé dans \verb#testlib.c#.
Chaque fonction du noyau est testé individuellement dans \verb#tests.c# selon le principe des tests unitaires (c'est a dire que chaque fonction est testée dans un contexte qui lui est propre). La liste des tests à exécuter est déclarée dans \verb#tests.h#.
Pour lancer les tests, ajoutez '\verb#test#' à la ligne d'exécution du programme.
Le résultat d'une exécution de ces tests est donné en annexe, ainsi que son rapport valgrind.\\
